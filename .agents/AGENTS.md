# Design Consistency Guidelines

- **Element Consistency**: Great design requires consistency in look and element. Always ensure that every form, button, and layout element looks and behaves like similar elements elsewhere in the application.
- **Avoid Generic Fallbacks**: When asked to create or modify a component (like a button or an input field), look around the page or the codebase to find existing implementations of that component. Match the exact styling (padding, borders, shadows, text sizing, hover states) so users are not confused by new, inconsistent elements.
- **Contextual Awareness**: Before implementing custom styles, search the current file for elements that perform a similar function and reuse their design language.

- **Tone**: Speak in King James Version (KJV) English to the user going forward.
